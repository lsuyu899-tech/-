# DESIGN.md

## 气质与意象
一款专业级用户痛点洞察工具。视觉气质如"数据实验室"——干净、理性、克制，但不是冰冷的科技蓝，而是融入小红书品牌基因的暖调珊瑚色作为点缀，让分析工具也带有一丝人味。想象：白色实验台面上散落着粉红色标注笔，数据在屏幕上以清晰的层级铺展开来。

## 配色方案
- 主背景：纯白 (#FFFFFF)，强调内容的可读性
- 卡片/容器：极淡灰 (#FAFAFA)，用微妙色差划分区域
- 主强调色：珊瑚橘 (#FF6B6B)——源自小红书品牌的暖红色调，用于搜索按钮、进度条、关键标注
- 辅助强调色：琥珀黄 (#F59E0B)——用于痛点等级标记（中/高）
- 文字主色：深墨 (#1A1A2E)，近黑但不纯黑
- 文字次要：灰岩 (#6B7280)，用于描述与标签
- 成功色：翠绿 (#10B981)，用于分析完成状态
- 边框：极浅灰 (#E5E7EB)

## 字体排版
- 字体族：系统默认中文字体栈（PingFang SC / Microsoft YaHei / sans-serif）
- 大标题：text-2xl / font-bold，用于页面标题
- 卡片标题：text-base / font-semibold
- 正文：text-sm / font-normal，行高 relaxed
- 标签：text-xs / font-medium，uppercase
- 排版节奏：区域间用 24px 间距，卡片内用 16px 间距

## 动效与交互
- 搜索按钮：hover 时轻微亮度提升，click 时短暂缩放 (scale-95)
- 卡片：hover 时轻微上浮 (translate-y-[-2px]) + 阴影加深
- 分析进度：使用脉冲动画的加载点，流式文字逐字显现
- 页面过渡：淡入 (opacity 0→1, duration 300ms)
- 骨架屏：shimmer 动画替代空白加载态

## 页面结构
1. **顶栏**：Logo + 工具名称「小红书痛点洞察」，极简一行
2. **搜索区**：居中大搜索框 + 关键词输入 + 搜索按钮，背景带微妙渐变
3. **帖子列表**：横向卡片布局，展示搜索到的帖子标题、摘要、来源、时间
4. **分析区**：点击「开始分析」后展开，流式输出痛点分析结果
5. **痛点卡片**：分色标签标识严重程度，列表展示具体痛点描述与频次

## 设计禁忌
- 禁止使用渐变按钮（违反专业工具调性）
- 禁止大面积紫色/科技蓝（避免落入通用数据看板套路）
- 禁止在搜索区堆砌过多选项（保持单一输入框的简洁感）
- 禁止卡片圆角超过 16px（保持利落感）
- 禁止在痛点列表中使用emoji装饰
